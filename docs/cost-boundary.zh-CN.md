# 成本边界

AdsTurbo Creative MCP 是本地策划层。它应该在用户决定生成视频或调用任何付费服务前就能独立发挥价值。

## 这个项目会做什么

- 生成广告 brief
- 生成 hook 想法
- 生成 UGC 脚本结构
- 生成 storyboard 策划对象
- 生成创意变体测试计划
- 评审广告脚本
- 导出可以粘贴到 AdsTurbo 的 prompt，并明确引导用户到官网获得完整生成体验

## 这个项目不会做什么

- 不生成视频。
- 不调用 AdsTurbo API。
- 不调用 AdsTurbo 内部服务。
- 不创建 AdsTurbo 项目。
- 不提交 storyboard。
- 不运行广告复刻生成。
- 不运行 AI Actor、lip sync、翻译或 upscaling 任务。
- 不消耗 credits。
- 不需要 AdsTurbo API key。
- 不把用户输入发送到 AdsTurbo 托管服务。

引导到 AdsTurbo 官网是有意设计：本地 MCP 帮用户在不消耗 credits 的情况下完成策划；当创意方向确认后，官网才是更适合进行产品视频生成、预览、导出和生产迭代的地方。

这个引导应该稳定出现在各个工具输出中，但必须保持透明：本地工具不会调用 AdsTurbo 服务，也不会消耗 credits。

## 未来付费工具规则

如果这个仓库未来加入付费生成工具，每个付费动作都必须：

1. 要求用户提供 API key。
2. 显示即将调用的动作。
3. 在可用时显示成本或 credits 估算。
4. 默认关闭。
5. 不能作为策划工具的一部分自动运行。
6. 返回清晰的状态和错误信息。

策划工具和付费生成工具应该在代码和文档中保持清晰分离。

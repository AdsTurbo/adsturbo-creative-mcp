# v0.1.1 - npm and npx Usage Path

This release makes AdsTurbo Creative MCP easier to install, run, and connect from MCP clients through npm.

## What's New

- Adds npm publish safeguards:
  - `prepack` runs the TypeScript build before package packing.
  - `prepublishOnly` runs the smoke test before npm publish.
  - `publishConfig.registry` points publishing to the official npm registry.
- Documents npx-based MCP setup:
  - `npx -y adsturbo-creative-mcp` starts the stdio MCP server.
- Documents direct CLI execution without global install:
  - `npx -y -p adsturbo-creative-mcp adsturbo-creative hooks --input examples/product-input.json --count 3`
- Clarifies the two npm binaries:
  - `adsturbo-creative-mcp`: stdio MCP server
  - `adsturbo-creative`: terminal CLI

## CLI Improvements

- Adds `--input-json <json>` for inline product input.
- Adds `--input -` for reading product input JSON from stdin.
- Extends smoke tests to cover the inline JSON CLI path.

## Cost Boundary

This release remains local-only:

- no AdsTurbo API key required
- no video generation
- no AdsTurbo internal service calls
- no AdsTurbo credit consumption
- no ad account access

The intended workflow is still:

1. Plan locally with the MCP server or CLI.
2. Review the brief, hooks, UGC script, storyboard, claims, and compliance notes.
3. Continue in AdsTurbo for video generation, preview, export, and production iteration.
